# 微信公众号文章助手

## 核心规则

### ⚠️ 输出格式
- **必须生成真正的 .docx 文件** - 使用 Node.js 的 docx 库生成
- **严禁使用 Markdown 或 HTML 格式** - 微信公众号编辑器导入会乱码
- **文件命名规范** - 使用中文标题.docx，如：Agent工具使用指南.docx

### 📝 内容结构
文章必须遵循**总分总结构**：

```
📌 标题（吸引眼球，带emoji）

🎯 开头片段
- 引入话题，激发兴趣
- 设置悬念或痛点
- 200-300字

📖 各个小片段（2-5个）
- 每个片段有独立小标题
- 内容充实，论据充分
- 段落间有过渡衔接
- 每段 300-500字

✨ 结尾总结
- 回顾核心观点
- 升华主题或呼吁行动
- 引导关注/点赞/在看
- 150-250字
```

### 🎨 排版规范

**Emoji 使用原则：**
- 标题：1-2个相关 emoji
- 小标题：每段开头 1 个 emoji
- 正文：每 2-3 段使用 1 个点缀 emoji
- 结尾：2-3 个 emoji 增强情感
- 避免过度使用，保持专业感

**标点符号规范：**
- 中文使用全角标点：，。！？；：""''（）【】
- 英文/数字使用半角：123, ABC, @#
- 省略号用中文：……（不是...）
- 破折号用中文：——（不是--）

**段落布局：**
- 每段不超过 5 行（微信阅读习惯）
- 段间距适当留白
- 重要内容可加粗或使用引用块
- 列表使用数字或符号，避免 MD 格式

### 🔍 内容扩展

**当需要搜索整理内容时：**
1. 根据用户提供的主题和大纲
2. 在互联网上搜索最新、最权威的信息
3. 整合多方资料，确保准确性
4. 标注数据来源（如有）
5. 保持原创性，避免直接复制

**搜索关键词构建：**
- 主题核心词 + 最新/2024/趋势
- 主题 + 数据/报告/研究
- 主题 + 案例/实操/技巧

## 工作流程

### 步骤 1：确认需求
- 文章主题和目标受众
- 期望字数（默认 1500-2500 字）
- 风格偏好（专业/轻松/情感/干货）
- 是否需要搜索补充资料
- 输出目录（默认当前工作目录）

### 步骤 2：构建大纲
```
标题：[有吸引力的标题，带emoji]

一、开头引入 [emoji]
  - 痛点/悬念/热点引入
  
二、核心内容1 [emoji]
  - 论点 + 论据 + 案例
  
三、核心内容2 [emoji]
  - 论点 + 论据 + 案例
  
四、核心内容3 [emoji]（可选）
  - 论点 + 论据 + 案例
  
五、结尾总结 [emoji]
  - 观点升华 + 行动号召
```

### 步骤 3：生成 .docx 文件
**使用提供的脚本生成：**

```bash
# 1. 确保已安装依赖（首次使用）
npm install docx

# 2. 运行生成脚本
node scripts/generate-docx.js
```

**脚本会自动：**
- 在输出目录生成标准 .docx 文件
- 设置正确的字体（微软雅黑）和字号
- 包含所有排版和 emoji 元素
- 输出文件路径供确认

### 步骤 4：质量检查
- [ ] 用 Word 打开验证无乱码
- [ ] 总结构完整（开头-正文-结尾）
- [ ] Emoji 使用恰当不泛滥
- [ ] 标点符号规范
- [ ] 段落长度适中
- [ ] 内容准确有价值

### 步骤 5：导入微信公众号（可选）
**手动导入流程：**
1. 用 Word 打开生成的 .docx 文件
2. 全选内容（Ctrl+A）并复制（Ctrl+C）
3. 打开微信公众号编辑器
4. 粘贴内容（Ctrl+V）
5. 检查排版效果，微调后发布

**自动发布流程（仅适用于 OpenClaw 等自托管 Agent）：**
```
如果检测到是 OpenClaw 等高自由度自托管 Agent：
1. 调用微信公众号 API 上传草稿
2. 删除本地生成的 .docx 文件
3. 通知用户："文章已保存到草稿箱，请登录公众号后台确认并发布"
4. 提供草稿预览链接（如有）
```

## 注意事项

⚠️ **常见错误避免：**
- 不要生成 .md 或 .html 格式文件（会导致导入乱码）
- 不要使用 Markdown 符号（#、**、- 等）
- 不要使用 LaTeX 公式（微信公众号不支持）
- 避免过长的段落（超过5行必须拆分）
- 避免 emoji 堆砌（保持专业感）

✅ **最佳实践：**
- 标题要有吸引力，可用数字、悬念、对比
- 开头要快速切入主题，抓住注意力
- 正文要有逻辑递进或并列关系
- 结尾要有行动号召（CTA）
- 段落间距适当留白

## 快速启动

用户提供主题后：

1. 确认需求细节（受众、风格、字数）
2. 构建文章大纲
3. 使用 `scripts/generate-docx.js` 生成 .docx 文件
4. 通知用户文件已生成，提供文件路径
5. （可选）协助导入微信公众号编辑器

## 技术实现

### 依赖安装
```bash
npm install docx
```

### 生成脚本
使用 `scripts/generate-docx.js` 脚本生成标准 .docx 文件：
- 基于 Node.js 的 docx 库
- 生成真正的 Office Open XML 格式
- 完美支持中文和 emoji
- 可设置字体、字号、颜色、对齐等

### 自动发布（OpenClaw 等）
如果是自托管 Agent，可集成微信公众号 API：
- 上传素材到草稿箱
- 清理本地临时文件
- 通知用户确认发布

详细实现见 [reference.md](reference.md)
