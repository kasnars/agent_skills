# Skill 开发规范指南

> 本指南教你如何编写一个规范、可被 Agent 正确加载和执行的高质量 Skill

---

## 📁 Skill 目录结构

每个 Skill 建议包含以下文件：

```
skill-name/
├── SKILL.md          # 核心配置文件（必须）
├── scripts/          # 脚本目录（可选）
│   └── run.sh
├── references/       # 参考文档（可选）
│   └── guide.md
└── assets/           # 资源文件（可选）
    └── logo.png
```

---

## 📝 SKILL.md 编写规范

### 1. 文件头（YAML Front Matter）

```yaml
---
name: skill-name
description: 一句话描述这个 skill 做什么，触发条件是什么
location: 自定义路径，不填则用默认路径
---
```

- **name**：Skill 名称，英文小写加下划线，唯一标识
- **description**：描述触发条件和功能，会被 Agent 扫描识别
- **location**：可选，指定加载路径

### 2. 触发词（Trigger Keywords）

列出所有触发此 Skill 的关键词或短语：

```
## 触发关键词

以下任意词/句触发本 skill：
- "关键词1"
- "关键词2"
- "完整句触发"
```

### 3. 执行流程（Action Steps）

用数字编号，每一步只做一件事，说完就等：

```
## 执行流程

### 第 1 步：步骤名称

具体操作内容...

**等用户回复后，继续第 2 步。**
```

**原则：**
- 每步只问一个问题
- 用"等用户回复"明确暂停点
- 不催，不连问

### 4. 核心行为准则（Guidelines）

列出 Agent 必须遵守的规则：

```
## 核心原则

- 全程用**第一人称**说话
- 每一步只做一件事
- 禁止说"好的，进入第X步"
- ...
```

### 5. 异常处理

```
## 异常处理

- **情况A** → 处理方式
- **情况B** → 处理方式
```

### 6. 绝对禁止

```
## 绝对禁止

- ❌ 禁止行为1
- ❌ 禁止行为2
```

---

## ✅ Skill 质量检查清单

编写完一个 Skill 后，对照检查：

- [ ] 有 YAML front matter（name + description）
- [ ] 有清晰的触发关键词
- [ ] 每步只有单一任务
- [ ] 暂停点明确（"等用户回复"）
- [ ] 异常情况有处理方案
- [ ] 没有绝对禁止的行为
- [ ] 用第一人称"我"而非第三人称
- [ ] 语言自然，不像机器人说话
- [ ] 注释和占位符说明清晰

## 🔧 推荐的 Skill 骨架模板

```yaml
---
name: example-skill
description: 简短描述触发条件和功能
---

# Example Skill

## 触发关键词

- "触发词1"
- "触发词2"

## 执行流程

### 第 1 步：做什么

说清楚要做什么...

**等用户回复后，继续第 2 步。**

### 第 2 步：做什么

...

## 核心原则

- 用第一人称"我"
- 每步只做一件事
- ...

## 异常处理

- **失败** → 说"出了问题，继续..."

## 绝对禁止

- ❌ 禁止行为
```

---

## 💡 注意事项

1. **不要预设固定顺序**：用户可能跳过中间步骤直接说最终目标
2. **接纳任何输入格式**：用户提供的信息可能不完整，理解意图而非死扣格式
3. **适时给予肯定**：用户配合时要表达感谢
4. **记录重要信息**：用户的偏好、设置等及时写入文件
5. **错误恢复**：出错了要继续推进而不是卡住
